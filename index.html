⚡ Database Triggers

8 automated triggers handle auto-calculations, cascades & backlog management — no manual code needed

TRIGGER 01
trg_attendance_calc

Before inserting attendance, auto-calculates attendance_percentage and sets eligibility_status — Eligible if ≥75%, else Not Eligible.

BEFORE INSERT ON attendance
TRIGGER 02
trg_faculty_insert

When a new faculty record is added, automatically inserts a row in faculty_subjects linking them to their assigned subject_code.

AFTER INSERT ON faculty
TRIGGER 03
trg_auto_student_subject

When a subject is assigned to faculty (in faculty_subjects), automatically enrols all existing students into that subject using INSERT IGNORE.

AFTER INSERT ON faculty_subjects
TRIGGER 04
trg_marks_calc

Before marks are inserted, auto-computes total_marks, assigns grade (S → F scale), grade_point (0.0–10.0), and sets Pass/Fail status.

BEFORE INSERT ON marks
TRIGGER 05
trg_backlog_on_marks_insert

After marks are inserted, if status = Fail, automatically creates a Pending backlog entry for that student–subject pair (no duplicates via IF NOT EXISTS).

AFTER INSERT ON marks
TRIGGER 06
trg_backlog_on_marks_update

Same logic as Trigger 05 but fires on marks updates — ensures backlogs are created if updated marks still result in a Fail status.

AFTER UPDATE ON marks
TRIGGER 07
trg_student_insert_auto

When a new student is added, automatically enrols them in all subjects already present in faculty_subjects (INSERT IGNORE prevents duplicates).

AFTER INSERT ON students
TRIGGER 08
trg_student_delete_auto

Before a student is deleted, removes all their rows from student_subject to maintain referential integrity and prevent orphan records.

BEFORE DELETE ON students

Step by Step

How to Setup the Database

Follow these steps carefully to get MySQL configured and the app running on your local machine.

1

Install MySQL & MySQL Workbench

Download and install MySQL 8.0 and MySQL Workbench from the official MySQL website. During MySQL installation, set a root password — you will need it in the next steps.

-- Official download links: https://dev.mysql.com/downloads/mysql/ https://dev.mysql.com/downloads/workbench/
2

Connect to MySQL via Workbench

Launch MySQL Workbench → click Local instance (localhost) → enter your root password → click OK to open the SQL query editor.

3

Create the Database

In the query editor, run the following SQL to create and select the database:

CREATE DATABASE student_management_db; USE student_management_db;
4

Run the Schema SQL Script

In Workbench go to File → Open SQL Script, select complete_database_schema.sql, then click ⚡ Execute. This creates all 9 tables and 3 views.

-- Or run from Windows Command Prompt: mysql -u root -p student_management_db < complete_database_schema.sql
5

Install the Triggers Script

Open and execute student_db_triggers.sql in the same way. This installs all 8 automated triggers.

-- Or run from Windows Command Prompt: mysql -u root -p student_management_db < student_db_triggers.sql
6

Configure the App's Database Connection

In the application's settings or configuration panel, enter the following connection details to connect Java to MySQL:

-- Connection settings to enter in the app Host : localhost Port : 3306 Database : student_management_db Username : root Password : <your MySQL root password> -- JDBC URL format used internally: jdbc:mysql://localhost:3306/student_management_db
7

Launch the Application

Double-click StudentApp-1.0.exe to start. Use the default credentials below to log in for the first time:

-- Default Admin Login Username : admin Password : admin123 -- Default Faculty Login Password : faculty123 -- Default Student Login (password = roll number) Password : <student roll number>
